# API Catalog Generation

`api_catalog.json` is generated by running the `generate_api_catalog.py` script, which operates on `services.json` and `service_function_ids.json`.

### Files
- **`services.json`**: The source file containing original APIs (no redundancy).
- **`service_function_ids.json`**: The file where function IDs are stored.
- **`api_catalog.json`**: The resulting catalog file.

### Script: `generate_api_catalog.py`
This script generates the `api_catalog.json` file by mapping APIs from `services.json` to their corresponding function IDs in `service_function_ids.json`. It creates an API entry for each `functionId`, grouping them by service name and tag (REST or SOAP).

#### Key Features:
- **Validation**: The script ensures all keys in `service_function_ids.json` exist in `services.json`. If a key is missing, it exits with an error message.
- **Matching Logic**: Matching occurs via exact or prefix matching.
    - *Example*: A service named "Service A" will match a function ID "Service A 1".
